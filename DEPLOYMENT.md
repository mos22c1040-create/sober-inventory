# رفع المشروع على استضافة (Deployment)

دليل خطوة بخطوة لرفع نظام المخزون على استضافة ويب (مثل cPanel أو استضافة مشتركة).

---

## ما تحتاجه

- استضافة تدعم **PHP 7.4+** و **MySQL**
- (اختياري) **HTTPS** — مطلوب تقريباً لمسح الباركود من كاميرا الجوال

---

## الخطوة 1: رفع الملفات

### الطريقة أ — رفع يدوي (FTP أو مدير الملفات)

1. ضغط المشروع كملف **ZIP** (بدون مجلد `skills` وبدون `.env`).
2. رفع محتويات المشروع إلى الاستضافة:
   - إن كان **جذر الموقع** = مجلد واحد (مثلاً `public_html`):
     - ارفع **كل** محتويات المشروع داخل هذا المجلد (مجلدات `app`, `config`, `public`, `storage`, `views` وملفات `.htaccess`, `README.md`...).
     - انسخ محتويات مجلد **`public`** إلى **جذر الموقع** (حيث يُفتح الموقع)، واترك باقي المجلدات (`app`, `config`, `storage`, `views`) في مستوى أعلى أو في مسار واحد معها حسب ما يسمح به الاستضافة.
   - الطريقة الأبسط التي تناسب معظم الاستضافات:
     - اجعل **جذر الموقع** يشير إلى مجلد **`public`** فقط (انظر الخطوة 4).
     - أو ارفع المشروع كاملاً ثم غيّر جذر الموقع إلى `public` إن كانت الاستضافة تسمح بذلك.

### الطريقة ب — عبر Git (إن كانت الاستضافة تدعمه)

من مجلد المشروع على جهازك:

```bash
git push origin main
```

على السيرفر (SSH أو Terminal الاستضافة):

```bash
cd /path/to/your/web/root
git clone https://github.com/mos22c1040-create/sober-inventory.git .
```

ثم تأكد أن جذر الموقع يشير إلى المجلد الذي فيه `public` أو إلى `public` نفسها (انظر الخطوة 4).

---

## الخطوة 2: إنشاء قاعدة البيانات

1. من **cPanel** (أو لوحة الاستضافة): ادخل إلى **MySQL Databases**.
2. أنشئ قاعدة بيانات جديدة، مثلاً: `mos22c10_inventory`.
3. أنشئ مستخدماً وأعطه صلاحية كاملة على هذه القاعدة.
4. سجّل: **اسم القاعدة**، **اسم المستخدم**، **كلمة المرور**، و**Host** (غالباً `localhost`).

---

## الخطوة 3: تنفيذ السكما وإنشاء المدير

1. من **phpMyAdmin** (أو أي أداة SQL في الاستضافة):
   - اختر قاعدة البيانات التي أنشأتها.
   - استورد الملف **`storage/schema.sql`** (من المشروع).
2. أضف مستخدماً مديراً (غيّر كلمة المرور لاحقاً):

```sql
INSERT INTO users (username, email, password, role, status)
VALUES (
  'admin',
  'admin@example.com',
  '$2y$10$92IXUNpkjO0rOQ5byMi.Ye4oKoEa3Ro9llC/.og/at2.uheWG/igi',
  'admin',
  'active'
);
-- كلمة المرور الافتراضية: password
```

## الخطوة 4: إعداد الملفات والإعدادات

### أ) ملف البيئة `.env`

- في **جذر المشروع** (المجلد الذي فيه `app` و `config` و `public`)، أنشئ ملفاً اسمه **`.env`** (أو عدّل إن وُجد).
- ضع فيه قيم الاستضافة (غيّر القيم حسب لوحة التحكم):

```env
DB_HOST=localhost
DB_PORT=3306
DB_DATABASE=mos22c10_inventory
DB_USERNAME=mos22c10_user
DB_PASSWORD=كلمة_مرور_قاعدة_البيانات
```

- تأكد أن ملف **`.env`** **لا يُرفع** إلى Git (موجود في `.gitignore`).

### ب) جذر الموقع (Document Root)

يجب أن تفتح الموقع عبر **`public/index.php`** فقط، لا عبر مجلدات أخرى.

- إن كانت الاستضافة تسمح بتعيين **Document Root**:
  - اجعلها تشير إلى مجلد **`public`** داخل المشروع.  
  مثال: إذا المشروع في `home/user/sober-inventory` فاجعل Document Root = `home/user/sober-inventory/public`.
- إن لم تستطع تغيير Document Root (مثلاً جذر الموقع ثابت مثل `public_html`):
  - ارفع محتويات **`public`** داخل جذر الموقع.
  - ارفع مجلدات `app`, `config`, `storage`, `views` وملف `.htaccess` الجذر إلى **مستوى أعلى** من جذر الموقع إن أمكن، أو انسخ محتوى **`index.php`** وعدّل فيه مسار `BASE_PATH` ليشير لمجلد المشروع.  
  (الطريقة الموصى بها: جعل Document Root = `public` لتجنّب التعقيد.)

### ج) صلاحيات المجلدات

- مجلد **`storage`** (وإن أنشأت داخله `storage/barcode_bridge` أو `storage/logs`) يجب أن يكون قابلاً للكتابة من قبل السيرفر (عادة 755 أو 775 حسب الاستضافة).

---

## الخطوة 5: التأكد من التشغيل

1. افتح عنوان الموقع في المتصفح، مثلاً: `https://yourdomain.com`
2. يجب أن تظهر صفحة تسجيل الدخول.
3. سجّل الدخول بـ:
   - البريد: `admin@example.com`
   - كلمة المرور: `password`
4. غيّر كلمة المرور من إدارة المستخدمين بعد أول دخول.

---

## استكشاف الأخطاء

| المشكلة | ما تفعله |
|--------|----------|
| صفحة بيضاء أو 500 | تفعيل عرض الأخطاء مؤقتاً في `public/index.php` أو مراجعة سجلات الخطأ في الاستضافة. |
| لا تظهر الصفحة الرئيسية | التأكد أن كل الطلبات تمر عبر `public/index.php` (إعادة كتابة الروابط / `.htaccess`). |
| خطأ اتصال بقاعدة البيانات | مراجعة `.env` (DB_HOST غالباً `localhost`)، والصلاحيات، ووجود القاعدة والمستخدم. |
| الجوال لا يفتح الكاميرا | الموقع يجب أن يعمل عبر **HTTPS** (شهادة SSL). |
